About the Project

This project is designed to bridge the gap between classroom learning and real-world software development workflows.

Project overview

What you'll learn

Students collaborate using Git by creating their own branches from a central repository. Each student enhances the frontend of an existing website by adding new features, improving UI/UX, or optimizing design.

This project simulates real-world development workflows, giving you hands-on experience with version control, code reviews, and collaborative development.

VERSION CONTROL

Master Git branching, commits, and pull requests in a real collaborative setting.

FRONTEND SKILLS

Apply HTML, CSS, and JavaScript to build real, functional improvements.

CODE REVIEW

Practice reviewing code and giving constructive feedback through pull requests.

Learning objectives

By the end of this project,
you will be able to:

🌿

Use Git branching effectively

Create feature branches, switch contexts, and merge code without breaking the main branch.

💻

Apply frontend dev skills

Write well-structured HTML, CSS, and JavaScript to build real improvements.

🎨

Improve UI/UX design

Enhance the visual design and user experience of an existing system.

🔍

Perform code reviews

Submit and review pull requests, giving and receiving constructive feedback.

⚡

Resolve merge conflicts

Understand and resolve basic merge conflicts that arise in collaborative development.

📝

Document your work

Write clear documentation covering features added, tools used, and challenges faced.

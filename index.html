The Git Workflow

Follow these 7 steps to complete the project from fork to pull request.

01
Fork Repo
Go to GitHub and fork the central repository to your account.
Click Fork (top-right)
→ Creates your copy
02
Clone Fork
Download your forked repo to your local machine.
git clone https://github.com/
theirusername/project.git
03
Create Branch
Make a personal feature branch with your last name.
git checkout -b feature/
<lastname>-frontend
04
Enhance UI
Implement at least 3 improvements to the frontend.
// Layout, animations,
// components, validation
05
Commit
Write meaningful commit messages for every change.
git add .
git commit -m "feat: ..."
06
Push Branch
Push your local branch up to your GitHub fork.
git push origin
feature/branch-name
07
Pull Request
Open a PR from your branch to the instructor's main branch.
Compare & pull request
→ base: main ✓

Required Enhancements

Implement at least 3 improvements — mix and match from any category below.

🎨
UI / UX Improvements
  • Improve layout spacing and alignment
  • Add responsive design (mobile-friendly)
  • Enhance typography and color palette
  • Add animations or transitions
⚙️
Functional Enhancements
  • Add interactive component (modal, dropdown)
  • Improve navigation (navbar, sidebar)
  • Implement form validation
  • Improve loading or feedback indicators
⚡
Optimization
  • Clean and organize your code structure
  • Reduce redundant CSS and JS
  • Improve basic performance metrics
  • Improve file and folder organization

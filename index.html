How It Works

Six steps to a
merged pull request.

Follow the same workflow used by professional engineering teams around the world.

01

Clone the Repository

Start by cloning the central repo to your local machine. This gives you the base project you'll be enhancing.

$git clone <repo-url> && cd <repo-name>
02

Create Your Feature Branch

Each student creates their own branch using their last name. Never work directly on main.

$git checkout -b feature/<lastname>-frontend-enhancement
03

Implement 3+ Frontend Improvements

Choose from UI/UX improvements, functional enhancements, or code optimization. Aim to impress — quality beats quantity.

04

Commit with Proper Messages

Follow conventional commit style. Keep each commit focused and meaningful — no "update file" commits allowed.

$git commit -m "feat: improved navbar responsiveness"
05

Push Your Branch

Push your feature branch to the remote repository so it's visible to your team and instructors.

$git push origin feature/<branch-name>
06

Open a Pull Request

Submit a PR to main with a summary of changes, before/after screenshots, and a list of features added. This is your final deliverable.

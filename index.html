The Process

Your 7-step workflow

Follow each step exactly as a professional developer would. No shortcuts.

01

Fork the Repository

Go to the instructor's repo on GitHub, click Fork (top-right). This creates your personal copy under your username.

github.com/aldonaire/frontend-enhancement-project → Fork
02

Clone Your Fork

Pull the forked repo to your local machine so you can work on it.

git clone https://github.com/yourusername/frontend-enhancement-project.git
03

Create Your Branch

Never work on main. Create a feature branch using your last name.

git checkout -b feature/lastname-frontend-enhancement
04

Implement 3+ Enhancements

Choose from UI/UX improvements, functional enhancements, or code optimization. Minimum three changes required.

// Improve layout, add animations, fix responsiveness...
05

Commit Properly

Write meaningful commit messages. Every commit must describe what changed and why.

git commit -m "feat: improved navbar responsiveness"
06

Push Your Branch

Push your feature branch to your forked repo on GitHub.

git push origin feature/lastname-frontend-enhancement
07

Open a Pull Request

Go to your fork on GitHub, click "Compare & pull request", and target the instructor's main branch.

base: aldonaire/main ← compare: yourusername/feature-branch
bash — ~/frontend-enhancement-project
$ git checkout -b feature/dela-cruz-frontend-enhancement
Switched to a new branch 'feature/dela-cruz-frontend-enhancement'
$ git add .
$ git commit -m "feat: add responsive navbar with hamburger menu"
[feature/dela-cruz] 3 files changed, 87 insertions(+), 12 deletions(-)
$ git push origin feature/dela-cruz-frontend-enhancement
Branch 'feature/dela-cruz' set up to track remote branch.

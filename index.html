Step-by-Step

Project Workflow

Follow these 7 steps to complete the project. Each step mirrors professional Git-based team workflows.

STEP 01
🍴

Fork the Repository

Go to the instructor's GitHub repo and click Fork (top-right). This creates your personal copy.

theirusername/frontend-enhancement-project
STEP 02
⬇️

Clone Your Fork

Pull your forked repo down to your local machine to start working.

git clone https://github.com/username/frontend-enhancement-project.git
STEP 03
🌿

Create a Branch

Create a feature branch using your last name. Never work directly on main.

git checkout -b feature/lastname-frontend-enhancement
STEP 04
🎨

Implement Enhancements

Add at least 3 improvements: UI/UX updates, functional features, or code optimization.

STEP 05
📝

Commit Properly

Use meaningful commit messages following the feat:, fix:, chore: convention.

git commit -m "feat: improved navbar responsiveness"
STEP 06
🚀

Push Your Branch

Push your local feature branch to your fork on GitHub.

git push origin feature/branch-name
STEP 07
🔀

Create a Pull Request

Go to your fork → click "Compare & pull request". Set base repo to aldonaire/frontend-enhancement-project, base branch main, and compare to your feature branch.

What You Must Build

Enhancement Requirements

Choose at least 3 improvements across any of these categories.

UI/UX

Interface Improvements

  • Improve layout — spacing and alignment
  • Add responsive design (mobile-friendly)
  • Enhance typography and color palette
  • Add CSS animations or transitions
Functional

Feature Additions

  • Add interactive component (modal, dropdown)
  • Improve navigation (navbar, sidebar)
  • Implement form validation
  • Better loading or feedback indicators
Optimization

Code Quality

  • Clean and organize HTML/CSS/JS
  • Remove redundant or unused code
  • Improve page performance
Deliverables

What to Submit

  • Your GitHub feature branch (pushed)
  • An open Pull Request to the base repo
  • README update or separate docs file
  • Features added, tools used, challenges encountered

⚠ RULES — READ CAREFULLY

  • → Do NOT push directly to main
  • → Work only on your assigned feature branch
  • → Keep commits meaningful — no "update file" messages
  • → Avoid breaking existing functionality
  • → Test your code before pushing

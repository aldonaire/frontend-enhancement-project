About the Project

Three Pillars of
Enhancement

Students collaborate using Git by creating their own branches from a central repository — each enhancing the frontend with new features, improved UI/UX, or optimized design.

01 UI/UX Improvements Required
Improve layout, spacing, and alignment. Add responsive design for mobile. Enhance typography and color usage. Add meaningful animations or transitions that improve user experience.
02 Functional Enhancements Required
Add interactive components like modals or dropdowns. Improve navigation with navbar or sidebar. Implement form validation and loading feedback indicators that guide the user.
03 Code Optimization Required
Clean and organize your code. Reduce redundant CSS and JavaScript. Improve performance with basic optimizations. Keep your commits meaningful and your branch history clean.

# Step-by-step Git workflow

$ git clone https://github.com/
theirusername/fe-project.git

$ cd fe-project
$ git checkout -b feature/lastname-frontend

# Make your enhancements...

$ git add .
$ git commit -m "feat: improved navbar"
$ git push origin feature/lastname-frontend

✓ Branch pushed — open a Pull Request

What to Submit

Your Deliverables

Three things must be submitted for a complete grade. Missing any one of them will reduce your score significantly.

01
GitHub Branch
Your feature branch must be pushed to your forked repository with all changes committed properly.
  • Branch named: feature/lastname-frontend-enhancement
  • All changes committed with meaningful messages
  • No direct pushes to main
02
Pull Request
Open a PR from your feature branch targeting the instructor's main branch. Fill in the PR description template.
  • Base: aldonaire/frontend-enhancement-project → main
  • Head: your fork → feature branch
  • PR description must include what was changed
03
Documentation (README update or separate file)
A short written document covering your work. This is worth 15 points — don't skip it.
  • Features added and what they do
  • Tools and libraries used
  • Challenges encountered
  • How you solved each challenge
  • Screenshots or a demo link (optional but great)
  • Time spent on the project

Project Rules
  • Push directly to main — always use a feature branch
  • Use vague commit messages like "update file"
  • Break existing functionality before pushing
  • Work only on your assigned branch
  • Test everything before pushing
  • Keep commits small and meaningful

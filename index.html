Workflow

Project Steps

Follow these 7 steps in order. Each one mirrors a real Git development cycle.

1

Fork the Repository

Go to the GitHub project repo and click Fork (top-right corner). This creates a personal copy of the repository under your own GitHub account.

# Your fork lives at:
github.com/yourname/frontend-enhancement-project
2

Clone Your Fork

Download your forked repository to your local machine so you can start coding in your editor.

git clone https://github.com/yourname/frontend-enhancement-project.git
cd frontend-enhancement-project
3

Create Your Branch

Never work on main directly. Create a feature branch using your last name in the branch name.

git checkout -b feature/<lastname>-frontend-enhancement
4

Implement Enhancements

Add at least 3 improvements from the categories listed below — UI/UX, functionality, or optimization. You can also attempt the optional bonus challenges.

5

Commit Your Changes

Write clear, descriptive commit messages. Every commit should explain what changed and why. No vague messages like "update file".

git add .
git commit -m "feat: improved navbar responsiveness"
git commit -m "feat: add modal component"
git commit -m "fix: corrected form validation"
6

Push Your Branch

Upload your local branch to your GitHub fork so it's available online and ready for a pull request.

git push origin feature/<branch-name>
7

Open a Pull Request

Go to your fork on GitHub and click Compare & pull request. Make sure you're targeting the correct base repository and branch.

base repo: aldonaire/frontend-enhancement-project
base branch: main
head repo: yourname/frontend-enhancement-project
compare: feature/<your-branch>
Requirements

Enhancement Categories

Pick at least 3 improvements from any category below. Mix and match freely.

🎨
UI / UX

Visual & Layout Improvements

  • Improve spacing, alignment, and visual hierarchy
  • Add responsive / mobile-friendly design
  • Enhance typography and color system
  • Add smooth animations or CSS transitions
⚙️
Functional

Interactive Functionality

  • Add interactive component — modal, dropdown, accordion
  • Improve navigation — navbar, sidebar, breadcrumbs
  • Add real-time form validation with feedback
  • Improve loading states and user indicators
⚡
Optimization

Code Quality & Performance

  • Clean and organize code structure
  • Remove redundant CSS and JavaScript
  • Improve basic performance — lazy loading, minify
  • Use semantic HTML for better accessibility
⭐
Bonus · Optional

Extra Credit Challenges

  • Implement a dark mode toggle
  • Integrate a public API — weather, quotes, etc.
  • Use a frontend framework like React or Vue

Rules

Guidelines

Follow these rules throughout the whole project.

01

Never push to main

All changes must go through a feature branch and a pull request.

02

Work only on your branch

Stay on your assigned feature branch throughout the entire project.

03

Write meaningful commits

Describe what changed and why. No "update file" or "changes" messages.

04

Don't break existing features

Your additions should improve the project — not break what already works.

05

Test before pushing

Always run your code locally and verify it works before pushing to GitHub.
